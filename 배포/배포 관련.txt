📌 무슨 이유로 Apache보다 Nginx를 더 사용하는 걸까?

그 이유는 바로 Nginx가 트래픽이 많은 웹 사이트에 더 적합하기 때문이다.

Nginx는 대용량 트래픽을 처리하기 위해 가벼움과 높은 성능을 목표로 하는 경량 서버이다.

'''
그런데 웹 서버는 정적 파일의 제공을 목적으로 사용하던 Apache를 사용하던 시기와 달리 최근에는 리버스 프록시, 로드 밸랜서, 메일 프록시 및 HTTP 캐싱 등 전체 범위에서 서버 작업을 처리하는 웹 서버로 발전하였다.
'''

다양한 기능과 클라이언트의 확장으로 대용량 트래픽을 처리해야하는 요즘 서비스는 무겁고 대용량 트래픽을 처리하기 어려운 Apache와는 맞지 않기 때문에 Nginx 를 채택하였다.

===========================================

웹서버 = nginx

웹서버(nginx)와 장고를 연결해주기 위해 gunicorn(wsgi) 사용
웹서버와 웹 어플리케이션을 연결해주는 것 = 즉 인터페이스 이다.
장고(웹 어플리케이션: 컴퓨터에서 돌아가는 프로그램), 
웹서버 : 외부에서 들어오는 request(요청)를 처리해주는 별도의 프로그램

원래는 장고가 static이랑 media를 제공해줬는데
nginx 사용하면 nginx가 static이랑 media 제공
용량이 크고 연산이 필요없는 고정적인 파일들이기에 바로 제공해줘
장고 성능 대폭 아낄 수 있고
대규모 트래픽도 처리
nginx와 장고는 별도의 기술
장고 내에도 wsgi 있어서 간단히 사용했는데
ec2에서는 제대로된 거 사용함
